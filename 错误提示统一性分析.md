# 错误提示统一性分析报告

## 📊 当前状态

### ✅ tmarks 项目 - 已有统一的 Toast 系统

**推荐使用方式：**
```typescript
import { useToastStore } from '@/stores/toastStore'

// 在组件中
const { addToast } = useToastStore()
addToast('error', '错误信息')
addToast('success', '成功信息')
addToast('info', '提示信息')
addToast('warning', '警告信息')

// 或者直接调用
useToastStore.getState().error('错误信息')
useToastStore.getState().success('成功信息')
```

## 🎯 统一方案建议

### 方案 1：全部改用 Toast（推荐）

**优点：**
- 统一的用户体验
- 非阻塞式提示
- 自动消失，不需要用户点击
- 支持多个提示同时显示

**缺点：**
- 对于需要用户确认的操作（如删除），需要额外的确认对话框

**实施：**
```typescript
// 替换 alert()
alert('错误信息') 
→ useToastStore.getState().error('错误信息')

// 替换 confirm() - 需要保留 ConfirmDialog 组件
if (confirm('确定删除吗？'))
→ 使用现有的 <ConfirmDialog> 组件
```

### 方案 2：混合使用（当前推荐）

**Toast 用于：**
- 成功提示
- 错误提示
- 信息提示
- 警告提示

**ConfirmDialog 用于：**
- 需要用户确认的操作（删除、批量操作等）

**AlertDialog 用于：**
- 重要的错误信息，需要用户明确知晓

## 🔧 具体修改建议

### 高优先级（影响用户体验）

1. **useTabGroupMenu.ts** - 替换所有 alert()
   - 简单提示 → Toast
   - 确认操作 → 需要重构为使用 ConfirmDialog

2. **TabGroupTree.tsx** - 替换 alert()
   ```typescript
   alert('重命名失败，请重试')
   → useToastStore.getState().error('重命名失败，请重试')
   ```

### 中优先级（代码一致性）

3. **TagManageModal.tsx** - 简化错误处理
   - 移除本地的 `showErrorAlert` 和 `errorMessage` 状态
   - 直接使用 `useToastStore`

4. **BatchActionBar.tsx** - 简化错误处理
   - 同上

### 低优先级（可选优化）

5. 为静默失败的地方添加用户提示

## 📝 tab 项目（浏览器扩展）

**当前方式：** 使用 Zustand store 的 `error` 和 `successMessage` 状态

**建议：** 保持当前方式，因为：
- 浏览器扩展空间有限
- 当前方式已经统一
- 不需要复杂的 toast 系统

**可选改进：** 可以添加一个简单的 toast 组件，但不是必需的。

---

## 🚀 实施步骤

### 第一步：修改 useTabGroupMenu.ts

这个文件有最多的 alert() 调用，需要：
1. 导入 `useToastStore`
2. 将简单的 alert() 替换为 toast
3. 对于 confirm()，需要重构为使用回调或状态管理

### 第二步：修改其他组件

逐个替换其他文件中的 alert() 和自定义 AlertDialog。

### 第三步：测试

确保所有错误提示都能正常显示，用户体验良好。

---

## ⚠️ 注意事项

1. **confirm() 的替换比较复杂**
   - 需要将同步的 confirm() 改为异步的对话框
   - 可能需要重构部分代码逻辑

2. **保持一致性**
   - 错误信息的措辞要统一
   - Toast 的持续时间要合理（错误 4-5 秒，成功 2-3 秒）

3. **用户体验**
   - 重要的错误可以保留 AlertDialog
   - 不要过度使用 toast，避免信息过载

- **自定义 AlertDialog：** 4 个组件
- **需要修改的文件：** 约 6-8 个
